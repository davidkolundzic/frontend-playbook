# Frontend Playbook

Personal playbook for structuring frontend work, patterns, desisions, and reusable snippets.

## Purpose

This repo helps me standardize how I approach:

- data fetching
- auth flow
- state management
- forms
- feature structure
- architecture decision

The goal is not to store everything.
The goal is to maintain:

- approved default patterns
- practical comparisons
- reusable snippets
- feature checklists
- working references

## How I use this repo

1. Start with `INDEX.md`
2. Check `06-decisions/approved-defaults.md`
3. Open a pattern note in `02-patterns/`
4. Jump to snippets in `03-snippets/`
5. Use checklists before starting a new feature
